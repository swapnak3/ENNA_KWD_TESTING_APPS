# README for enna_kwd_testing


## Introduction

This project aims to enable automated testing of a machine readable testcase specification as so called keyword driven 
testing. 
All test specifications which are based on the psw keyword specification shall be enabled to be tested automated.
Every specified keyword must be realized by a fitting enna stimulation.



## Usage 

### Prerequisites
- Python 3.13
- Access to ENNA resources and artifactory on cariad github or vw devstack

**Important**: Please use a virtual environment to install the requirements. And uninstal standsrd enna_hcp_configuration

```
pip install -r requirements.txt
pip uninstall enna_hcp_configuration
```

Entry point to run the campaign ist the campaign file:
```
python -m enna_kwd_testing.campaigns.campaign
```

The generic campaigns structure is as following:

```
# -*- coding: utf-8 -*-
"""@project: enna_kwd_testing.

Contains start for generic campaign which is generated by .json source.
"""

import enna.core.campaign
import enna.core.campaign_runner

import enna_kwd_testing.campaigns.alignments.alignment
import enna_kwd_testing.factories.base_config
import enna_kwd_testing.utilities.stimulation_factory_generator.generator
from enna_kwd_testing import ROOT_PATH

SOURCE_FILE = ROOT_PATH / "enna_kwd_testing/campaigns/json_input/dummy_testsequence.json"
TEMPLATE_FILE = ROOT_PATH / "enna_kwd_testing/utilities/stimulation_factory_generator/templates/template.jinja"
DEPENDENCY_FACTORY_PATH = ROOT_PATH / "enna_kwd_testing/factories/generated_dependency_factory.py"

if __name__ == "__main__":
	camp = enna.core.campaign.Campaign("keyword_driven_testing_prototype")
	tests = camp.add_testgroup(SOURCE_FILE.name)
	enna_kwd_testing.utilities.stimulation_factory_generator.generator.generate(source_file=SOURCE_FILE, output_path=DEPENDENCY_FACTORY_PATH, template_path=TEMPLATE_FILE)
	tests.add_testcase_factory(enna_kwd_testing.factories.base_config.BASE_FACTORY_PATH)
	enna.core.campaign_runner.start_campaign(camp, enna_kwd_testing.campaigns.alignments.alignment.DEFAULT_ALIGNMENT)
```

This will run the campaign in the .json input file which is referenced in:
```
enna_kwd_testing.campaigns.campaign.SOURCE_FILE
```

The .json input must fit the following structure to be able to run the campaign:
[Input sheme](#input-sheme)

When the campaign starts, the generic dependencies must be initialized. 
Therefor, an analyzer browses all the given keyword stimulations and builds a basic stimulation factory containing
the keyword stimulations as RequiredStimulations. Base for the build is the jinja-template, referenced in 
```
enna_kwd_testing.campaigns.campaign.TEMPLATE_FILE
```

The resulting, dependency factory file will appear in the path, reference in:
``
enna_kwd_testing.campaigns.campaign.DEPENDENCY_FACTORY_PATH
``
Now the stimulation factory, used by the campaign file will inherit from the dependency factory file and so be able to
access all the necessary components which were initialized for the testrun.

**To better understand the dependency workflow see also:**
[Dependency workflow](\xternal_res\flow_enna_factory.png)


## Results

1. An error in any precondition stimulation results in: **SKIPPED**
2. An error in any action stimulation results in: **ERROR**
3. An error in any expected result stimulation results in: **FAILURE**
4. An error in any postcondition stimulation results in a **WARNING**


## Contributing

Current project path is:
``
<PSW_GIT>/ENNA_KWD_TESTING
``

### Prerequisites:
- Python 3.11
- Access to ENNA resources and artifactory on cariad github or vw devstack

### Guideline:
- Any keyword which shall be automated must be realized as a stimulation to ensure that components can be initialized 
dynamically
- The automated keyword stimulation must be referenced in the current keyword specification excel work sheet and later 
in planned specification database
- The keyword stimulation reference must start from source root 
(e.g.: enna_kwd_testing.stimulations.actions.susan.susan_terminal.SwitchCl15)
- The keyword stimulations must be categorized as actions or results
- The keyword stimulations must return ```True``` or ```False```, indicating whether the stimulation failed or not
- Keyword stimulation can use other keyword stimulations if necessary and depending on the given specification

### Supporting scripts:

#### Important -> Please be aware of the codestlye! Also feel free to use the enna ide_tools if preferred
1. **run_pylint.bat**: Checks the codestyle for consistency and correctness as specified in internal coding guidelines

2. **build_dev.bat**: Build a wheel and source distribution and output it to ``<PROJECT_ROOT>/dist/``
3. **build.bat**: Build a wheel and source distribution and output it to ``<PROJECT_ROOT>/dist/``. Also generate a new
branch to backup the current state with it´s current version name specified in ``version.py``. 
(Branchname: "enna_kwd_testing_v_MAJOR_MINOR_BUG-OR-TWEAK")
